# AGENTS.md

## Code Style
- Use Black for Python formatting.
- Avoid abbreviations in variable names.
- Always check existing types before creating new ones

## Testing
- Run tests before finalizing a PR.
- All commits must pass lint checks via flake8.

## PR Instructions
- Title format: [Fix] Short description
- Include a one-line summary and a "Testing Done" section
- Generate docstrings
- Create PR with unit tests
- Post copyable unit tests in a comment
- Commit unit tests in the feature branch
